# Project Overview

---

## Purpose

The main purpose of this project is to **simplify and automate the recruitment process** between **students** and **recruiters**.  
It provides a platform that eliminates manual communication and creates an efficient, transparent process for applying and hiring.

The system ensures:
- Faster communication between job seekers and recruiters.
- Centralized data storage for all job and application records.
- Secure and easy management of user data and job listings.

---

## Technology Stack

- **Frontend:** React.js, HTML5, CSS3, JavaScript (with Vite and TailwindCSS for styling)
- **Backend:** Node.js, Express.js
- **Database:** MongoDB (Cloud via MongoDB Atlas)
- **Authentication:** JWT (JSON Web Token) based role management
- **Hosting:** Vercel (Frontend) and Render (Backend)
- **Version Control:** Git & GitHub for branch-based development

---

## Key Features

- **Secure Authentication:** Only verified users can log in and perform specific actions.
- **Authorization:** Role-based permissions for students and recruiters.
- **Recruiter Dashboard:** Allows job posting, editing, and managing applicants.
- **Student Dashboard:** Enables job browsing, applying, and profile management.
- **Resume Upload:** Students can upload and update resumes anytime.
- **Application Tracking:** Recruiters can update application status, and students can view statuses.
- **Responsive Design:** Works smoothly across all devices.

---

## Project Highlights

This project demonstrates how real-world job portals work using the **MERN stack**.  
It includes complete CRUD functionality, API integration, and database relationships between users, jobs, and applications.

It also serves as a base project that can be further extended to include:
- Admin dashboards
- Notification systems
- Job recommendations using AI
